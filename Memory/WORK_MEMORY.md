# 工作记忆

## 项目用途

这个项目用于学习计算机和 AI 相关知识，长期记录刷题过程、题目解析、错题、知识点总结和协作工作流。

## 当前协作约定

- 用户会发送牛客、力扣、知识星球等平台的题目。
- 助手需要选择正确选项，并用大白话解释。
- 用户可能会给出自己的理解；如果理解错误，要纠正。
- 用户如果明确说“答案是 X”或“官方答案是 X”，应识别为提供答案或纠正信息。
- 平台答案不一定绝对正确，需要客观看待。
- 每次出现新工作流、工作流调整、有价值知识点、题目解析时，要更新相关文档。
- 当前默认学习专题是大模型概念。用户如果没有明确切换专题，之后的题目默认按大模型概念处理并归档到 `LLMConcept/NOTES.md`。
- 用户如果开始学习其他板块，会明确说明，例如 Agent、提示词工程、RAG、Prompt 判断、微调等；届时再切换归档目录。
- 用户要求：展示数学公式时使用专门、直观的展示方式，优先用 LaTeX 块公式和分步推导。

## 文档结构

- 根目录 `STUDY_WORKFLOW_PROMPT.md`：长期提示词和工作流。
- `Memory/WORK_MEMORY.md`：沟通记录、阶段成果、当前约定。
- 各知识目录下的 `NOTES.md`：记录题目解析与知识点。
- `Notes/`：暂时无法归类或通用内容。
