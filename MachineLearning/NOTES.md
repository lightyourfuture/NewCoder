## 2026-06-13 精确率 Precision 的定义

来源：用户提供题目

题目：
> 在模型评估中，什么是“精确率”（Precision）？

选项：
- A. 正确预测的样本数与总样本数的比率
- B. 正确预测的正样本数与所有预测为正样本的比率
- C. 正确预测的正样本数与所有正样本数的比率
- D. 模型在验证集上的损失值

结论：
正确选项：B

大白话解析：
精确率看的是：模型预测为正样本的这些结果里，有多少是真的正样本。它关注的是“模型报正的时候准不准”，也就是减少误报。

公式：

\[
\text{Precision}
=
\frac{TP}{TP + FP}
\]

其中：

\[
TP = \text{真正例，预测为正且实际为正}
\]

\[
FP = \text{假正例，预测为正但实际为负}
\]

易错点：
- A 是准确率 Accuracy。
- C 是召回率 Recall。
- D 是损失值 Loss，不是精确率。

知识点总结：
- Precision = 预测为正的样本中，有多少是真的正。

## 2026-06-13 处理不平衡数据集的方法

来源：用户提供题目

题目：
> 在数据预处理阶段，如何处理不平衡的数据集？

选项：
- A. 忽视不平衡问题
- B. 增加少数类样本
- C. 删除少数类样本
- D. 只关注多数类样本

结论：
正确选项：B

大白话解析：
不平衡数据集指某些类别样本很多，某些类别样本很少。模型如果直接训练，容易偏向多数类，看起来准确率高，但少数类识别很差。常见处理方法是增加少数类样本，比如过采样、数据增强、SMOTE 等。

补充：
- 除了增加少数类样本，也可以减少多数类样本，或使用类别权重、重采样、调整损失函数等方法。
- 但在本题选项里，最合理的是 B。

易错点：
- 忽视不平衡会让模型偏向多数类。
- 删除少数类样本会让少数类信息更少，问题更严重。
- 只关注多数类会导致少数类召回率很差。

知识点总结：
- 类别不平衡：常用少数类过采样、数据增强、SMOTE、类别权重等方法处理。

## 2026-06-13 无监督学习的定义

来源：用户提供题目

题目：
> (2025_美团)在AI领域，什么是“无监督学习”？

选项：
- A. 需要标注数据进行训练
- B. 不依赖于标签进行数据分析
- C. 仅用于分类任务
- D. 只适用于小规模数据集

结论：
正确选项：B

大白话解析：
无监督学习就是数据没有人工标好的答案，模型要自己从数据里找规律。比如把相似用户分成几类、发现数据中的异常点、把高维数据压缩成低维表示等。

易错点：
- A 是监督学习，需要标注数据。
- C 错。无监督学习不只用于分类，常见任务是聚类、降维、异常检测等。
- D 错。无监督学习也可以用于大规模数据。

知识点总结：
- 无监督学习 = 不依赖标签，让模型从数据自身结构中发现规律。

## 2026-06-13 ROC 曲线的定义

来源：用户提供题目

题目：
> 在模型评估中，什么是“ROC曲线”？

选项：
- A. 反映模型训练时间的曲线
- B. 描述模型准确率的曲线
- C. 描述真正率和假正率的关系
- D. 显示模型的超参数设置

结论：
正确选项：C

大白话解析：
ROC 曲线用于评估二分类模型在不同分类阈值下的表现。它的横轴是假正率，纵轴是真正率。通过改变阈值，可以看到模型在“尽量找出正样本”和“尽量少误报负样本”之间的权衡。

公式：

\[
TPR = \frac{TP}{TP + FN}
\]

\[
FPR = \frac{FP}{FP + TN}
\]

其中：
- \(TPR\)：真正率，也叫召回率，表示真实正样本中有多少被正确识别。
- \(FPR\)：假正率，表示真实负样本中有多少被错误预测为正。

易错点：
- ROC 不是训练时间曲线。
- ROC 不只是准确率曲线，它看的是 TPR 和 FPR 的关系。
- ROC 不显示超参数设置。

知识点总结：
- ROC 曲线 = 不同阈值下，真正率 TPR 与假正率 FPR 的关系曲线。

## 2026-06-13 准确率 Accuracy 的定义

来源：用户提供题目

题目：
> 在模型评估中，什么是“准确率”？

选项：
- A. 正确预测的样本数与总样本数的比率
- B. 召回率和精确率的调和平均
- C. 正确预测的正样本数与所有正样本数的比率
- D. 模型在验证集上的损失值

结论：
正确选项：A

大白话解析：
准确率看的是模型在所有样本里一共答对了多少。无论正样本还是负样本，只要预测对了都算正确。

公式：

\[
\text{Accuracy}
=
\frac{TP + TN}{TP + TN + FP + FN}
\]

其中：
- \(TP\)：真正例，预测为正且实际为正。
- \(TN\)：真负例，预测为负且实际为负。
- \(FP\)：假正例，预测为正但实际为负。
- \(FN\)：假负例，预测为负但实际为正。

易错点：
- B 是 F1 分数。
- C 是召回率 Recall。
- D 是损失值 Loss。
- 数据类别严重不平衡时，准确率可能 misleading，需要结合 Precision、Recall、F1 等指标。

知识点总结：
- Accuracy = 所有样本中预测正确的比例。

## 2026-06-14 过拟合 Overfitting 的定义

来源：用户提供题目

题目：
> (2025_美团)在模型训练中，何为“过拟合”？

选项：
- A. 模型在训练集和验证集上表现良好
- B. 模型在训练集上表现良好，但在验证集上表现差
- C. 模型训练时间过长
- D. 模型参数设置不当

结论：
正确选项：B

大白话解析：
过拟合就是模型把训练集“背得太熟”，甚至连训练数据里的噪声、偶然规律也学进去了。所以它在训练集上表现很好，但一到没见过的验证集或真实数据上，表现就明显变差。

典型表现：
- 训练集准确率很高；
- 验证集准确率低，或者验证集 loss 变高；
- 模型泛化能力差。

易错点：
- A 是理想情况，不是过拟合。
- C 训练时间过长可能导致过拟合，但它不是过拟合的定义。
- D 参数设置不当可能引发过拟合，但也不是定义本身。

知识点总结：
- Overfitting = 训练集表现好，验证集/测试集表现差，本质是泛化能力不足。

## 2026-06-14 模型验证的主要目的

来源：用户提供题目

题目：
> 在大模型的应用中，"模型验证"的主要目的是？

选项：
- A. 提高模型的准确性
- B. 确保模型在未见数据上的表现
- C. 降低计算速度
- D. 简化模型的结构

结论：
正确选项：B

大白话解析：
模型验证的目的不是让模型在训练数据上看起来更好，而是检查它遇到没见过的新数据时表现怎么样。因为真实应用里，模型面对的大多不是训练集原题，而是新用户、新样本、新场景。

易错点：
- A 不准确。验证本身不直接提高准确性，它是评估和发现问题的过程。
- C 错。降低计算速度不是验证目标。
- D 错。简化结构属于模型压缩或架构优化，不是验证的主要目的。

知识点总结：
- Model Validation = 用未见数据评估模型泛化能力，判断模型能不能可靠用于真实场景。

## 2026-06-14 可解释性模型的主要优点

来源：用户提供题目

题目：
> 可解释性模型的一个主要优点是什么？

选项：
- A. 更高的预测能力
- B. 易于调试和优化
- C. 更低的计算成本
- D. 更少的参数

结论：
正确选项：B

大白话解析：
可解释性模型的主要优点是能看懂模型为什么做出某个预测。这样当模型出错时，更容易定位问题，比如是某个特征影响太大、数据有偏、规则不合理，还是模型学到了错误规律。

易错点：
- A 不一定。可解释性强不代表预测能力一定更高，有时复杂黑盒模型预测能力更强。
- C 不一定。可解释性和计算成本不是一回事。
- D 不一定。参数少可能更容易解释，但不是可解释性模型的主要优点。

知识点总结：
- Interpretability = 让模型决策过程更容易被理解，从而方便调试、优化、审计和建立信任。

## 2026-06-14 超参数 Hyperparameter 的定义

来源：用户提供题目

题目：
> 在模型训练过程中，什么是“超参数”？

选项：
- A. 模型在训练过程中自动学习的参数
- B. 训练数据集中的特征
- C. 在训练前手动设置的参数
- D. 模型评估时使用的指标

结论：
正确选项：C

大白话解析：
超参数是在训练开始前由人或搜索算法设定的配置，不是模型自己从数据里学出来的。比如学习率、batch size、训练轮数、正则化强度、树的深度、神经网络层数等，都属于超参数。

参数和超参数的区别：
- 参数：模型训练过程中学到的，比如神经网络权重 \(W\)、偏置 \(b\)。
- 超参数：训练前设定的，比如学习率、batch size、epoch 数。

易错点：
- A 是模型参数，不是超参数。
- B 是特征，不是超参数。
- D 是评估指标，比如 Accuracy、F1、AUC，不是超参数。

知识点总结：
- Hyperparameter = 训练前设置、控制训练过程或模型结构的配置项。
