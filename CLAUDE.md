# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## 仓库概述

这是一个 AI/计算机知识学习笔记仓库，用于长期积累牛客、力扣、知识星球等平台的刷题记录。仓库本身不是代码项目——没有构建、测试或 lint 命令。交互方式是你向用户讲解题目、解析知识点，并维护笔记文档。

## 核心工作流

### 回答题目的固定流程

收到题目后，严格按以下顺序回复：

1. 先给结论：正确选项是什么。
2. 大白话解释：为什么它对。
3. 逐个分析易混淆选项：为什么其他选项错。
4. 如果题目或答案有争议：说明争议点，给出客观判断。
5. 总结成一句记忆口诀或关键知识点。

### 用户输入的类型识别

- 用户问"选什么/为什么/这个对吗" → 按题目分析回答。
- 用户明确说"答案是 X"或"官方答案是 X" → 这是用户在提供/纠正答案，不是普通提问。优先当作约束或待核验信息处理。
- 用户的理解如果正确 → 明确肯定并补充更严谨表述。
- 用户的理解如果错误 → 温和但明确纠正，指出错在概念、条件还是陷阱。

### 批量题目处理规则

用户一次性发多题时（如 30 题随机组卷），必须逐题处理：
- 同一轮中按题号从前到后，每题单独给完整解析。
- 对话框回复不能压缩成"选项：一句话"的答题卡式摘要，每题都要写"正确答案：**X**" + 大白话解释 + 易错项分析 + 归档位置。
- 同时将每题完整写入对应的笔记文件。

## 笔记归档规则

### 根目录文件

- `STUDY_WORKFLOW_PROMPT.md`：长期协作提示词和工作流定义。工作流有调整时更新这里。
- `Memory/WORK_MEMORY.md`：沟通记录、当前约定、阶段成果、待办。每次有价值的交互都要在此追加记录。

### 知识归档目录

按题目领域归档到对应目录的 `NOTES.md`：

| 目录 | 内容范围 |

| `DataAnalyse/` | 数据分析、统计、业务分析、SQL |

| `DeepLearning/` | 深度学习、神经网络、优化器、GAN |
| `LLMConcept/` | 大模型概念、Transformer、微调、量化、推理优化 |

## 当前协作偏好

- 使用中文回答。
- 解释要像给初学者讲清楚一样，少堆术语。保留必要专业词，但要翻译成直观含义。
- 先讲结论，再讲为什么。
- 展示数学公式时优先使用 LaTeX 块公式和分步推导，避免纯文本堆公式。
- 回答时要方便用户顺着追问继续展开。
- 平台答案不一定绝对正确，需要客观看待，可以指出"平台答案可能不严谨/有争议"。
